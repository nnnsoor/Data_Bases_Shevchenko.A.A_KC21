# Практична робота №7: Створення фізичної моделі у MS SQL Server

## Опис проекту

Реалізація фізичної моделі бази даних для системи управління фермерським господарством у середовищі MS SQL Server.

## Мета роботи

- Реалізація логічної моделі даних у вигляді фізичної моделі
- Створення таблиць з коректним визначенням типів даних
- Використання обмежень цілісності для забезпечення достовірності даних
- Встановлення зв'язків між таблицями через зовнішні ключі

## Структура бази даних

### Основні таблиці:
- **Brigade** - інформація про бригади
- **Worker** - персональні дані працівників
- **Land_Plot** - характеристики земельних ділянок
- **Crop** - види сільськогосподарських культур
- **Equipment** - сільськогосподарська техніка
- **Fertilizer** - види добрив
- **Worker_Equipment_Skill** - навички працівників (many-to-many)
- **Fertilizer_Application** - внесення добрив на ділянки

## Бізнес-правила

Реалізовано 20 бізнес-правил:
- 5 обмежень доменної цілісності (CHECK)
- 4 обмежень сутнісної цілісності (PRIMARY KEY)
- 4 обмежень посилальної цілісності (FOREIGN KEY)
- 3 обмежень унікальності (UNIQUE)
- 4 обмежень обов'язковості (NOT NULL)

## Функціональні можливості

### Запити:
- Визначення площі під культури
- Виведення робітників бригади з технікою
- Пошук культури з максимальною врожайністю
- Пошук бригад з технікою понад середню
- Список ділянок з невідповідними добривами

### Транзакції:
- Купівля техніки
- Найм/звільнення працівників

## Технології

- **СУБД**: Microsoft SQL Server
- **Мова**: T-SQL
- **Інструменти**: SQL Server Management Studio (SSMS)
