## База даних

**`Farm_Management_Shevchenko`** — система управління фермерським господарством.

### Предметна область

На фермі є кілька земельних ділянок з унікальними номерами. Кожна ділянка характеризується площею, наявністю зрошення та посіяною культурою. Ділянки обслуговуються бригадами, які складаються з працівників із різними навичками керування сільськогосподарською технікою.

### Таблиці (8 шт.)

| Таблиця | Опис |
|---|---|
| `BRIGADE` | Бригади (бригадир, дата формування) |
| `CROP` | Культури (врожайність, потреба у зрошенні) |
| `FERTILIZER` | Добрива (тип, склад NPK, норма внесення) |
| `LAND_PLOT` | Ділянки (площа, якість ґрунту, культура, бригада) |
| `WORKER` | Працівники (ПІБ, дата найму, телефон, бригада) |
| `EQUIPMENT` | Техніка (тип, модель, рік випуску, статус) |
| `WORKER_EQUIPMENT_SKILL` | Навички керування технікою (рівень, сертифікація) |
| `FERTILIZER_APPLICATION` | Внесення добрив (дата, кількість, спосіб) |

### Зв'язки

### Завдання 3: Агрегатні функції

**Запит 3.1 — Площа ділянок по бригадах** (`SUM`, `COUNT`, `AVG`, `MIN`, `MAX`, `HAVING`)  
Загальна, середня, мінімальна та максимальна площа ділянок для кожної бригади. Фільтрація бригад без ділянок через `HAVING COUNT > 0`.

**Запит 3.2 — Якість ґрунту в розрізі культур** (`AVG`, `MIN`, `MAX`, `COUNT`)  
Середній індекс якості ґрунту та розкид значень по кожній культурі. Виключення NULL через `WHERE soil_quality_index IS NOT NULL`.

**Запит 3.3 — Статистика внесень за типом добрива** (`COUNT`, `SUM`, `COUNT DISTINCT`)  
Кількість внесень, загальний обсяг (кг) та кількість унікальних ділянок для кожного типу добрива (мінеральне / органічне / органо-мінеральне).

---

### Завдання 4: Віконні функції

**Запит 4.1 — `ROW_NUMBER()`**  
Ранжування працівників за стажем у межах кожної бригади. `PARTITION BY brigade_id` скидає нумерацію для кожної бригади. Найрання дата найму = ранг 1.

**Запит 4.2 — `SUM() OVER()`**  
Наростаючий підсумок внесень добрив по ділянці. Використано фрейм `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW` для класичного running total.

**Запит 4.3 — `RANK()`, `DENSE_RANK()`, `NTILE()`**  
Ранжування ділянок за якістю ґрунту. Демонструє різницю між `RANK` (пропускає ранги при рівних значеннях) та `DENSE_RANK` (без пропусків). `NTILE(4)` ділить ділянки на 4 квартилі.

### Завдання 5: Рядкові функції

**Запит 5.1 — Форматування ПІБ** (`UPPER`, `LOWER`, `CONCAT`, `LEN`, `REPLICATE`)  
Прізвище у верхньому регістрі, ім'я у нижньому, повне ПІБ з умовним по батькові, маскування прізвища зірочками для GDPR-звітів.

**Запит 5.2 — Аналіз телефонних номерів** (`LEFT`, `CHARINDEX`, `SUBSTRING`, `REPLACE`)  
Виділення коду оператора, частини після дефіса, нормалізація номера (видалення дефісів), підрахунок цифр.

**Запит 5.3 — Розбір марок техніки** (`UPPER`, `TRIM`, `LEFT`, `CHARINDEX`, `LTRIM`, `SUBSTRING`)  
Нормалізація типу техніки, виділення торгової марки та коду моделі з рядка.

---

### Завдання 6: Функції обробки дати

**Запит 6.1 — Стаж працівників** (`DATEDIFF`, `GETDATE()`, `CASE`)  
Стаж у роках, місяцях і днях. Категоризація: `Новачок` (< 1 р.), `Стажист` (1–4 р.), `Досвідчений` (5–9 р.), `Ветеран` (≥ 10 р.).

**Запит 6.2 — Аналіз внесень по місяцях** (`DATEPART`, `DATENAME`, `FORMAT`)  
Групування внесень добрив за роком, місяцем і кварталом. Кількість внесень, загальний обсяг, перша та остання дата у кожному місяці.

**Запит 6.3 — Контроль строків сертифікацій** (`DATEADD`, `DATEDIFF`, `FORMAT`, `CASE`)  
Дата закінчення сертифікату (3 роки від отримання), кількість днів до закінчення, статус: `ПРОСТРОЧЕНО` / `ЗАКІНЧУЄТЬСЯ СКОРО` / `Дійсна`.
